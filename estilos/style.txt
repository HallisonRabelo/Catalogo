Aula: Conexão PHP com Banco de Dados (MySQLi)
1. O que este código faz?
Este script tem uma função vital: estabelecer uma comunicação entre o servidor onde o PHP está rodando e o servidor de banco de dados MySQL. Ele utiliza a extensão MySQLi (MySQL Improved), que é uma das formas mais comuns e simples de lidar com bancos de dados no PHP moderno.

O fluxo consiste em definir as credenciais de acesso, tentar a conexão e verificar se houve algum erro que impeça o funcionamento do sistema.

2. Análise Detalhada do Código
Aqui está o seu código comentado linha a linha para você entender o papel de cada instrução:

PHP

<?php

// --- PASSO 1: Definição das Credenciais ---
// Armazenamos os dados do servidor em variáveis para facilitar a manutenção.
$host = "localhost";    // O endereço do servidor (geralmente local)
$usuario = "root";      // Nome do usuário do banco de dados
$senha = "Admin@123";   // Senha de acesso
$banco = "catalogo";    // Nome do banco de dados específico que queremos usar

// --- PASSO 2: Tentativa de Conexão ---
// A função mysqli_connect abre uma nova conexão com o servidor MySQL.
$conexao = mysqli_connect($host, $usuario, $senha, $banco);

// --- PASSO 3: Verificação de Erros ---
// O '!' antes da variável $conexao significa "se NÃO houver conexão"
if (!$conexao) {
    // A função die() interrompe a execução do script e exibe uma mensagem.
    // mysqli_connect_error() nos diz exatamente qual foi o problema técnico.
    die("Falha na conexão: " . mysqli_connect_error());

} else {
    // Se passar pelo teste acima, a conexão foi bem-sucedida.
    echo "Conectado com sucesso";
}

?>
3. Notas do Professor (Dicas Extras)
Segurança: Em um projeto real, nunca deixe suas senhas visíveis no código principal. Costumamos usar arquivos de configuração (.env ou config.php) protegidos.

O "die": A função die() é drástica. Ela para tudo. Em sistemas em produção, é mais elegante redirecionar o usuário para uma página de erro amigável.

MySQLi vs PDO: O mysqli que usamos aqui é excelente para iniciantes e projetos específicos para MySQL. Se você pretender trocar de banco de dados no futuro (ex: para PostgreSQL), aprenderemos mais tarde o PDO.
